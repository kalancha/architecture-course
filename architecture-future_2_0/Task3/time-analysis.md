# TIME-классификация текущих систем


| Система | Техническая пригодность | Функциональная пригодность | TIME | Решение |
|---|---|---|---|---|
| DWH на SQL Server 2008 | Низкая: устаревшая версия, сотни ТБ, отчёты часами, смешение функций | Высокая: критичные данные и логика | **Migrate** | Извлекать бизнес-логику в домены; данные переносить через CDC и parallel run; выводить части после переключения |
| PowerBuilder operator UI | Низкая: legacy и зависимость от старого DWH | Пока высокая: необходим клиникам | **Migrate → Eliminate** | Перенести функции в поддерживаемое клиническое приложение; удалить только после функционального паритета |
| Кастомизированный BI поверх DWH | Средняя/низкая из-за связанности | Высокая: требуется отчётность | **Migrate** | Сохранить Power BI как клиент, но перенести семантику и наборы на curated views Lakehouse |
| Центральный Apache Camel ESB | Низкая для целевого масштаба и независимости | Высокая в текущем ландшафте | **Migrate** | Переносить маршруты по одному на доменные API/события; запретить новые маршруты |
| Python AI services | Средняя/высокая после проверки версий | Высокая | **Invest** | Изолировать медицинский/ИИ-контур, добавить SLO, observability и автоматизированное развёртывание |
| Java/Go fintech services | Средняя/высокая после проверки версий | Высокая | **Invest** | Развивать независимо, усилить отказоустойчивость, финансовую сверку и data contracts |
